# Game Bug Report

> 作者：罗飞扬
> 测试平台：移动端（主）/ PC端
> 涉及游戏：鸣潮（Wuthering Waves）、崩坏：星穹铁道

---

## 鸣潮（Wuthering Waves）

---

### WW-001 · Critical

**标题：** 煌陇地图全息战略·哀声鸷——光噪层数≥7时角色全键位锁定，相关文字说明缺失

| 字段 | 内容 |
|------|------|
| 版本 | 1.x |
| 严重等级 | Critical |
| 测试平台 | 移动端已复现，PC端待验证 |
| 缺陷类型 | 机制说明缺失 / 操作异常 |

**复现路径：**
1. 进入1.x版本煌陇地图全息战略·哀声鸷
2. 持续承受光噪debuff，累积至≥7层
3. 观察角色操作响应

**预期结果：**
角色保持正常操作，或Boss说明/光噪buff描述中有明确文字提示该机制

**实际结果：**
角色所有按键被锁定无法操作，Boss说明与光噪buff描述均未提及此机制

**缺陷分析：**
机制说明缺失，玩家无法预判，易误判为游戏崩溃或操作bug，建议在光噪buff说明及Boss技能描述中补充相关文字提示

---

### WW-002 · Minor

**标题：** 2.5版本隐海试验场地图边界存在缝隙，可通过钩锁或飞翔技能穿出地图外

| 字段 | 内容 |
|------|------|
| 版本 | 2.5 |
| 严重等级 | Minor |
| 测试平台 | 移动端及PC端已复现 |
| 缺陷类型 | 碰撞体缺失 / 地图边界异常 |

**复现路径：**
1. 进入2.5版本隐海试验场地图
2. 靠近地图边界区域
3. 使用钩锁功能或角色自带飞翔技能朝边界方向移动

**预期结果：**
地图边界碰撞体完整，角色无法穿出地图范围

**实际结果：**
存在缝隙，角色可穿越边界到达地图外部

**缺陷分析：**
碰撞体设置不完整，边界封闭性未通过完整测试，可能导致角色卡死在地图外，已在移动端及PC端复现

---

### WW-003 · Minor

**标题：** 3.1版本水边使用大龟显像后跳向龟背，角色直接沉入水底而非触发游泳状态

| 字段 | 内容 |
|------|------|
| 版本 | 3.1 |
| 严重等级 | Minor |
| 测试平台 | 移动端及PC端已复现 |
| 缺陷类型 | 状态机切换异常 / 碰撞体交互异常 |

**复现路径：**
1. 进入3.1版本含水域场景地图
2. 在水边使用大龟显像技能
3. 朝着显像大龟背部方向跳跃
4. 观察角色落水后状态

**预期结果：**
角色落水触发游泳姿态，正常游泳

**实际结果：**
角色直接沉入水底，未触发游泳状态切换

**缺陷分析：**
大龟显像与水域边界碰撞体交互异常，状态机切换逻辑未覆盖该场景，已在移动端及PC端复现

### WW-004 · Critical

**标题：** 深塔副本Boss概率性逃出限制区域及掉落平台后满血复位

| 字段 | 内容 |
|------|------|
| 版本 | 不限 |
| 严重等级 | Critical |
| 复现概率 | 偶发 |
| 测试平台 | 移动端及PC端已复现 |
| 缺陷类型 | AI寻路异常 / 血量重置逻辑异常 |

**复现路径：**
1. 进入深塔副本场景
2. 与Boss持续战斗
3. 观察Boss移动路径及平台边缘行为

**预期结果：**
Boss始终在限制区域内活动

**实际结果一：**
Boss概率性移动至限制区域外，玩家无法攻击，战斗陷入僵局

**实际结果二：**
Boss概率性掉落平台后满血在初始位置复位，之前造成的伤害全部归零

**缺陷分析：**
Boss寻路AI边界约束逻辑存在漏洞，掉落判定与血量重置逻辑异常，直接阻断战斗流程，已在移动端及PC端复现

### WW-005 · Major

**标题：** 莫宁重击被飞廉之猩召唤的小猴子拦截，无法命中Boss导致协奏能量无法正常积累

| 字段 | 内容 |
|------|------|
| 版本 | 不限 |
| 严重等级 | Critical |
| 测试平台 | 移动端及PC端已复现 |
| 缺陷类型 | 碰撞体与伤害判定不一致 |

**复现路径：**
1. 使用莫宁对战飞廉之猩
2. 等待飞廉之猩召唤小猴子
3. 对小猴子与Boss之间的位置使用莫宁重击
4. 观察协奏能量变化

**预期结果：**
重击穿透无受击判定的召唤物，正常命中Boss并积累协奏能量

**实际结果：**
小猴子碰撞体拦截重击，攻击未命中Boss，协奏能量无法正常获取，影响战斗循环节奏

**缺陷分析：**
小猴子存在碰撞体但无受击判定，建议排查所有含召唤物Boss对空战角色的交互逻辑，已在移动端及PC端复现

### WW-006 · Minor

**标题：** 1.x版本全息战略·哀声鸷抱起角色飞行时退出战斗，角色保持被抱起姿态

| 字段 | 内容 |
|------|------|
| 版本 | 1.x |
| 严重等级 | Minor |
| 测试平台 | 移动端及PC端已复现 |
| 缺陷类型 | 动画状态机异常 / 退出逻辑未重置状态 |

**复现路径：**
1. 进入1.x版本全息战略·哀声鸷
2. 等待哀声鸷执行抱起角色飞向高空的技能
3. 在角色被抱起飞行过程中退出战斗
4. 观察退出后角色姿态

**预期结果：**
退出战斗后角色恢复正常站立姿态

**实际结果：**
角色仍保持被抱起的飞行姿态，动画状态未重置

**缺陷分析：**
战斗退出逻辑未处理Boss技能过程中的角色状态重置，动画状态机存在边界遗漏，已在移动端及PC端复现

## 崩坏：星穹铁道（Honkai: Star Rail）

---

### HSR-001 · Minor

**标题：** 进入战斗场景后角色头部骨骼小概率异常旋转，退出战斗后恢复正常

| 字段 | 内容 |
|------|------|
| 版本 | 不限 |
| 严重等级 | Minor |
| 复现概率 | 偶发 |
| 测试平台 | 移动端及PC端已复现 |
| 缺陷类型 | 骨骼动画异常 |

**复现路径：**
1. 进入任意战斗场景
2. 观察角色模型头部表现
3. 小概率触发头部持续旋转
4. 退出战斗观察是否恢复

**预期结果：**
角色头部骨骼动画正常，无异常旋转

**实际结果：**
角色头部出现异常旋转，战斗全程持续，退出战斗后自动恢复

**缺陷分析：**
骨骼绑定或动画状态机在战斗场景切换时存在概率性异常，已在移动端及PC端复现
